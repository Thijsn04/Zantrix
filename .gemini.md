# Zantrix EPD - Gemini Agent Instructions

*Always read this file at the start of each session when working on the Zantrix project.*

## 1. Project Context
Zantrix is an open-source, modular Electronic Patient Record (EPD) designed to compete with enterprise systems like EPIC and HiX. It uses a **Spring Modulith (Java 21+) backend** with PostgreSQL and Elasticsearch, and a **React (TypeScript, Vite) frontend** using Tailwind CSS.
The architecture is FHIR-native, meaning HL7 FHIR R4 is the core data model.

## 3. Strict Rules for AI Assistance
1. **Never leak PHI**: No Protected Health Information (even mocked PII if it looks real) should be logged without masking.
2. **Strict TypeScript**: Do not use `any`. Use proper interfaces for all frontend models.
3. **Robust Backend**: Always use `GlobalExceptionHandler` for errors. Ensure all new DB operations are tracked via `@AuditLoggable`.
4. **Testing First**: Use `IntegrationTestBase` (with Testcontainers) for integration tests. Do not use H2 databases.

## 4. Current Workflow
We are focusing on completing the MVP fundament (Modules 1.1 through 3.1). Any new feature must fit within the Modulith architecture without breaking strict domain boundaries.
